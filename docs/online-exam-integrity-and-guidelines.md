# AcademiQ 온라인 시험 — 부정행위 방지 및 응시 주의사항

> **문서 목적**: 현재 AcademiPlatform에 **실제 구현된** 온라인 시험 기능을 기준으로, 부정행위로 간주·감지·기록되는 행위와 수험자·관리자가 알아야 할 주의사항을 정리합니다.  
> **기준 일자**: 2026-06-07  
> **관련 코드**: `frontend/app/(public)/exam/`, `backend/src/online-exam/`

---

## 1. 개요

AcademiQ 온라인 시험은 다음 흐름으로 운영됩니다.

1. **접수·승인** — 시험 접수 후 관리자가 `온라인 응시 승인`을 해야 로비 입장 가능
2. **대기실(로비)** — 응시 시간·환경(웹캠·전체화면) 사전 점검
3. **응시 화면** — 제한 시간 내 답안 작성·자동 저장·제출
4. **감독·채점** — 관리자가 실시간 감독·이벤트 확인 후 채점·결과 공개

부정행위 방지는 **사전 환경 검사**, **응시 중 행위 로깅**, **웹캠 스냅샷**, **관리자 실시간 모니터링**을 조합합니다.  
다만 **경고 횟수에 따른 자동 실격·자동 무효 처리**는 현재 버전에 **구현되어 있지 않으며**, 운영자가 이벤트·스냅샷을 검토해 판단합니다.

## 2. 응시 자격 및 시간 규칙 (시스템 강제)

아래 조건을 만족하지 않으면 시험을 **시작할 수 없습니다**.

| 항목 | 설명 | 구현 위치 |
|------|------|-----------|
| 접수 상태 | `접수 완료(APPLIED)` | `getLobby()` |
| 온라인 승인 | `examEligibility = APPROVED` | 관리자 접수자 목록 |
| 온라인 접수 | 접수 시 `examMode = ONLINE` 또는 회차가 ONLINE/HYBRID | `getLobby()` |
| 게시된 시험지 | 회차에 `PUBLISHED` 시험지 존재 | `getLobby()` |
| 응시 시간 | `examWindowStart` ~ `examWindowEnd` 내 (상시 모의고사는 창 없음) | `resolveExamSchedule()` |
| 1회 응시 | 동일 접수 건으로 **이미 제출된 시험**이 있으면 재시작 불가 | `startAttempt()` |
| 웹캠 사전 확인 | 회차 `requireWebcam=true` 시 로비에서 권한 확인 필수 | 로비 UI |
| 전체화면 사전 확인 | 회차 `requireFullscreen=true` 시 로비에서 전체화면 전환 필수 | 로비 UI |

### 응시 시간 계산

- **개인 제한 시간**: 시험 시작 시점 + `durationMinutes` (기본 60분)
- **회차 종료 시각**: `effectiveWindowEnd` (응시 창 종료)
- **실제 만료 시각**: 위 둘 중 **더 이른 시각** (`expiresAt`)
- 만료 시 프론트엔드가 **자동 제출**을 시도합니다.

### 문항·보기 순서

- 시험지 설정에 따라 **문항 순서 셔플** (`shuffleQuestions`)
- **보기 순서 셔플** (`shuffleOptions`)
- 시작 시점의 문항·보기는 `questionSnapshot` / `optionsSnapshot`으로 고정 저장되어, 이후 문제은행이 바뀌어도 해당 응시 세션에는 영향 없음

## 3. 응시 중 시스템이 감지·기록하는 행위

감독 이벤트(`ExamProctorEvent`)는 응시 화면에서 서버로 전송됩니다.  
`HEARTBEAT`를 제외한 대부분의 이벤트는 `warningCount`(경고 카운트)를 **1 증가**시킵니다.

### 3.1 현재 프론트엔드에서 실제 기록되는 이벤트

| 이벤트 타입 | 감지 조건 | 화면에서의 제한 | 경고 카운트 | 부정행위 성격 |
|-------------|-----------|-----------------|-------------|---------------|
| `TAB_BLUR` | 브라우저 탭이 숨김 상태 (`document.hidden`) — 다른 탭 전환, 창 최소화 등 | 없음 (기록만) | +1 | **높음** — 시험 화면 이탈 |
| `COPY_ATTEMPT` | 복사(`copy`) 시도 | **복사 차단** (`preventDefault`) | +1 | **높음** — 답안·지문 유출 시도 |
| `PASTE_ATTEMPT` | 붙여넣기(`paste`) 시도 | **붙여넣기 차단** | +1 | **높음** — 외부 답안 반입 시도 |
| `FULLSCREEN_EXIT` | 전체화면 해제 (`fullscreenchange`) | 없음 (기록만) | +1 | **중간~높음** — 집중 환경 이탈 |
| `WEBCAM_PERMISSION_DENIED` | `requireWebcam`인데 카메라 권한·스트림 실패 | 웹캠 미리보기 없음 | +1 | **높음** — 신원·감독 거부 |
| `WEBCAM_SNAPSHOT` | 웹캠 캡처 후 스냅샷 API 업로드 성공 시 | 없음 (정상 절차) | +1 ※ | 감독 증빙 (※아래 참고) |

**※ `WEBCAM_SNAPSHOT` 주의**: 스냅샷은 3분 주기 정상 감독 절차이나, 현재 구현상 `warningCount`도 함께 증가합니다. 관리자 화면의 “이벤트 N건”은 **부정행위만이 아니라 스냅샷 건수도 포함**할 수 있으므로, 해석 시 이벤트 **타입별 목록**을 함께 확인해야 합니다.

### 3.2 차단만 하고 별도 이벤트는 남기지 않는 행위

| 행위 | 처리 |
|------|------|
| 우클릭(컨텍스트 메뉴) | `contextmenu` **기본 동작 차단** (`preventDefault`) |
| | 스키마에 `CONTEXT_MENU` 타입은 있으나, **현재 프론트엔드에서 해당 이벤트를 서버에 기록하지 않음** |

### 3.3 스키마에 정의되었으나 현재 미연동인 이벤트

다음 타입은 DB·API에 정의되어 있으나, **응시 화면에서 아직 발생시키지 않습니다**.

| 이벤트 타입 | 의도(설계) | 현재 상태 |
|-------------|------------|-----------|
| `HEARTBEAT` | 생존 신호·연결 유지 | 미전송 (전송 시 경고 증가 없음) |
| `HEARTBEAT_MISSED` | 하트비트 누락 | 미구현 |
| `WINDOW_FOCUS_LOST` | 창 포커스 상실 | 미구현 (`TAB_BLUR`만 유사 감지) |
| `CONTEXT_MENU` | 우클릭 시도 | 차단만, 미기록 |
| `NETWORK_RECONNECT` | 네트워크 재연결 | 미구현 |
| `FORCE_SUBMIT` | 관리자/시스템 강제 제출 | 미구현 |

## 4. 웹캠 감독 (requireWebcam=true 회차)

회차 설정에서 **웹캠 필수**가 켜진 경우:

### 로비(시작 전)

- `getUserMedia({ video: true })`로 **카메라 권한·동작 확인**
- 확인 완료 전에는 **시험 시작 버튼 비활성화**

### 응시 중

- 전면 카메라 스트림 유지 (화면 우하단 미리보기)
- **시작 직후 1회** + **3분마다** canvas 캡처 → JPEG 업로드
- 저장 경로: 서버 `uploads/proctor-snapshots/` (관리자 감독 화면에서 썸네일 조회)

### 부정행위로 해석되는 웹캠 관련 행위

- 응시 중 카메라 가리기·다른 사람 대리 응시·화면 이탈
- 권한 거부 또는 스트림 중단 (`WEBCAM_PERMISSION_DENIED`)
- 스냅샷에 복수 인원·무관한 화면·휴대폰 사용 등 (관리자 육안 검토)

---

## 5. 전체화면 (requireFullscreen=true 회차)

### 로비(시작 전)

- `document.documentElement.requestFullscreen()` 성공 여부 확인
- 실패 시 시험 시작 불가

### 응시 중

- **전체화면 유지를 강제하는 코드는 없음** (로비에서 한 번 확인 후, 응시 화면 진입 시 자동 재진입 없음)
- 응시 중 전체화면을 **해제하면** `FULLSCREEN_EXIT` 이벤트 기록

→ 운영상 “전체화면 필수” 회차는 수험자에게 **응시 내내 전체화면 유지**를 주의사항으로 안내해야 합니다.

## 6. 답안·제출 규칙 (부정행위 연계)

| 규칙 | 설명 |
|------|------|
| 자동 저장 | 문항 변경 시 `PATCH /answers`로 서버 저장 |
| 제출 후 수정 불가 | `IN_PROGRESS`가 아니면 답안 변경 API 거부 |
| 타인 응시 불가 | `attempt.userId`와 로그인 사용자 일치 검증 |
| 시간 종료 후 | `assertMutableAttempt` 경로에서 만료 시 자동 제출 처리 |
| 서술형·파일형 | 텍스트 직접 입력 또는 **외부 URL 제출** (파일 업로드 검증 없음) |

### 기술적으로 막히지 않는 행위 (운영·규정으로 금지)

- **다른 기기·브라우저·사람의 대리 응시** (본인 계정 공유)
- **검색 엔진·AI·메신저·교재** 등 외부 정보 참조 (탭 이탈은 `TAB_BLUR`로 일부 감지)
- **휴대폰·두 번째 모니터** 사용 (전용 감지 없음)
- **서술형 답안의 타인 작성·대필** (유사도·AI 검사 미구현)
- **파일 제출 URL의 사후 교체** (제출 시점 URL만 저장)
- **스크린샷·촬영**으로 문항 유출 (복사는 차단, 촬영은 미차단)

## 7. 관리자 감독·사후 검토

### 감독 화면 (`/admin/exam/[id]/proctor`)

- 회차별 **응시 중·완료 응시자** 목록
- **warningCount** (이벤트 누적)
- 최근 **proctorEvents** (타입·시각)
- **proctorSnapshots** 썸네일
- SSE 스트림 + 10초 폴링으로 갱신

### 접수자 목록 (`/admin/exam/[id]/applications`)

- 온라인 답안 제출 상태
- `warningCount` 표시 (예: “경고 3”)

### 무효 처리 (`INVALIDATED`)

- DB·UI에 `INVALIDATED` 상태·`invalidatedReason` 필드 존재
- **경고 N회 시 자동 무효화 로직은 현재 없음**
- 필요 시 관리자가 운영 정책에 따라 **수동 판정** (향후 API·UI 확장 가능)

## 8. 부정행위 유형별 정리 (운영 관점)

아래는 AcademiQ 온라인 시험에서 **금지되며**, 기술적 근거·감독 자료로 추적 가능한 대표 사례입니다.

### 8.1 확실히 감지·기록되는 행위

1. **시험 화면 이탈** — 다른 탭·앱으로 전환 (`TAB_BLUR`)
2. **복사 시도** — 문항·답안 복사 (`COPY_ATTEMPT`, 차단됨)
3. **붙여넣기 시도** — 외부 텍스트 반입 (`PASTE_ATTEMPT`, 차단됨)
4. **전체화면 해제** — 전체화면 필수 회차에서 해제 (`FULLSCREEN_EXIT`)
5. **웹캠 거부·중단** — (`WEBCAM_PERMISSION_DENIED` + 스냅샷 부재)

### 8.2 감독 자료로 사후 판단하는 행위

1. **대리 응시·이상 행위** — 웹캠 스냅샷에서 타인 출현·본인 불일치
2. **부정 자료 참조** — 이벤트 다발 + 답안 패턴·시간 분석 (수동)
3. **문항 유출** — 복사 차단 외 촬영·외부 공유 (규정 위반, 기술 미감지)
4. **동일 답안 공유** — 여러 응시자 간 유사 답안 (수동·통계 미구현)

### 8.3 시스템상 허용되지만 부정행위에 해당하는 행위

- 접수 계정을 타인에게 제공
- 응시 시간 내 가족·동료와 구두 상담
- 허용되지 않은 참고서·인터넷 검색 (이탈 흔적과 병행 판단)
- 파일 제출형 문항에 타인이 작성한 결과물 URL 제출

## 9. 수험자 주의사항 (응시 전·중·후)

### 9.1 응시 전 준비

- [ ] **본인 계정**으로 로그인했는지 확인 (계정 공유·대리 응시 금지)
- [ ] 안정적인 **유선 또는 Wi-Fi** 환경 준비
- [ ] 노트북 **전원 연결** 권장 (배터리 방전으로 중단 방지)
- [ ] **조용하고 독립된 공간** 확보 (타인 출입 최소화)
- [ ] 회차 안내에 따라 **신분 확인** 가능한 상태 유지
- [ ] `requireWebcam` 회차: 웹캠 정상 동작·조명·얼굴 노출 확인
- [ ] `requireFullscreen` 회차: 전체화면 전환 가능한 브라우저 사용 (Chrome·Edge 권장)
- [ ] 시험 시작 가능 **시간대** 확인 (상시 모의고사는 승인 후 즉시 가능)
- [ ] 마이페이지에서 **온라인 응시 승인** 완료 여부 확인

### 9.2 응시 중 필수 준수 사항

- [ ] 시험 화면을 **끝까지 유지** — 불가피한 이탈 최소화 (이탈 시 기록됨)
- [ ] **복사·붙여넣기 금지** (시도 시 차단 및 기록)
- [ ] **우클릭 메뉴 사용 금지**
- [ ] **전체화면 유지** (필수 회차)
- [ ] **웹캠 가리기·끄기 금지** (필수 회차, 3분마다 촬영)
- [ ] **휴대폰·태블릿·두 번째 모니터** 사용 금지
- [ ] **검색·AI 챗봇·메신저·교재** 등 외부 도움 금지
- [ ] **타인과의 대화·질의** 금지
- [ ] 답안은 **직접 작성** (서술형·파일 제출 포함)
- [ ] 문항 이동 후 답이 **자동 저장**됨 — “저장됨” 표시 확인
- [ ] 남은 시간(**타이머**) 확인 — 종료 시 **자동 제출**
- [ ] 제출 버튼은 **신중히** — 제출 후 **수정 불가**

### 9.3 응시 후

- [ ] 마이페이지 **시험 접수 내역**에서 완료·채점·결과 상태 확인
- [ ] 결과는 관리자 **결과 공개** 후 조회 가능 (공개 전에는 “결과 대기”)
- [ ] 이의 제기·문의는 운영 안내 채널(1:1 문의·공지) 이용

### 9.4 권장 환경

| 항목 | 권장 |
|------|------|
| 브라우저 | Chrome, Edge 최신 버전 |
| OS | Windows 10+, macOS 12+ |
| 네트워크 | 10Mbps 이상 안정 연결 |
| 주변 | 본인만 있는 실내, 조명 충분 |

## 10. 제재·운영 정책 (권장)

현재 시스템은 **자동 실격**을 수행하지 않습니다. 아래는 운영 시 권장 기준 예시입니다 (기관 정책에 맞게 조정).

| 수준 | 예시 | 권장 조치 |
|------|------|-----------|
| 경미 | 전체화면 1회 해제 후 즉시 복귀 | 경고 안내·기록 유지 |
| 중대 | `TAB_BLUR`·`COPY_ATTEMPT` 반복, 스냅샷 이상 | 답안 검토 강화·감점 또는 재시험 불가 |
| 중대 | 웹캠 거부·대리 응시 정황 | **응시 무효(`INVALIDATED`)** 검토 |
| 최중대 | 문항 유출·조직적 부정 | 시험 무효·자격 취소·법적 조치 검토 |

모든 판정은 **이벤트 로그·스냅샷·제출 시각·답안 내용**을 종합해 관리자가 결정합니다.

## 11. 구현 한계 및 향후 개선 (참고)

문서화 목적상, 현 버전에서 **아직 없는** 기능도 명시합니다.

- 경고 횟수 임계값 도달 시 **자동 제출·자동 무효**
- 응시 중 **전체화면 강제 유지** 및 미유지 시 즉시 경고
- `WINDOW_FOCUS_LOST`, `HEARTBEAT` 기반 이탈 감지
- **다중 모니터·원격 데스크톱·VM** 탐지
- **락다운 브라우저** 연동
- 서술형 **표절·AI 생성 탐지**
- 문항 화면 **워터마크·인쇄 방지** 강화
