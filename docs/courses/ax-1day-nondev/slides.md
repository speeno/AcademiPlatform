# AX 입문 1Day
## 하루 만에 시작하는 바이브 코딩
하네스 & Agent Skills 맛보기 — 비개발자용

> "코딩 몰라도, 오늘 'AI로 만든다'를 경험한다."

---

## 오늘의 목표
- AX·바이브 코딩이 무엇인지 **내 말로** 설명
- AI에게 **잘 시키는 법**
- **동작하는 미니 도구 1개** 완성
- 하네스·`SKILL.md` 맛보기 + 내 업무 적용

---

## 바이브 코딩이란?
- 코드를 짜는 게 아니라 **"무엇을·왜"를 설명**하면 AI가 만든다
- 🚗 나 = 운전대(의도·검증) / AI = 엔진(코드)
- 🍳 요리를 못 해도 **정확히 주문**하면 셰프가 만든다 — 맛은 내가 본다

---

## 잘 시키는 법 — 4요소
1. **무엇을**: "한 페이지 체크리스트 웹앱"
2. **왜/맥락**: "회의 준비용", "한국어로"
3. **예시/입력**: 샘플·양식을 붙인다
4. **제약**: "단순하게", "외부 전송 금지"

> 안 되면: "더 단순한 방법으로 다시 해줘."

---

## 랩 A — 첫 미니 도구
- 할 일 체크리스트 웹앱 만들기(또는 계산기/요약기)
- 한 단계씩: 뼈대 → 추가 기능 → 체크 표시 → 다듬기
- 완성 기준: **화면에서 동작** + 스크린샷 📸

---

## 하네스 맛보기
- 하네스 = AI가 **스스로 파일을 만들고·실행·수정**하는 환경(Cursor)
- 3요소: 도구(손) · 맥락(눈) · 루프(반복)
- 변경은 **수락(Apply)** 후 실행해 확인

---

## 랩 B — 한 단계 키우기
- 내 도구에 기능 1개 추가
- 예) "새로고침해도 유지", "완료 개수 표시"
- 맥락을 주고 **작게, 한 번에 하나**

---

## Agent Skills 맛보기
- 반복 업무 절차를 `SKILL.md`로 적어두면 **매번 같은 품질**
- 평소엔 설명만, 필요할 때 절차를 펼쳐 따른다
- `description`("언제 쓰나")이 가장 중요

---

## 안전 수칙
- ❌ 개인정보·기밀·비밀번호 금지
- ✅ 샘플/가공 데이터만
- ✅ 결과는 **사람이 검증**

---

## 내 업무에 적용하기
- "내 불편 1개 → 가장 작은 버전(MVP) 한 줄"
- 예) "매주 정산 엑셀 합치기를 붙여넣기 한 번으로"

---

## 마무리
- **산출물**: 동작하는 미니 도구 + 적용 아이디어 1줄
- **더 배우려면**: N3(3일) · N7(1주, 팀 프로젝트 완주)
- **회고**: 오늘 가장 놀라웠던 순간
