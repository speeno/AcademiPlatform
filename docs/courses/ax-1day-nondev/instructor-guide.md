# [N1] AX 입문 1Day — 강사 운영안

> 하루로 "두려움 제거 → 첫 성공 → 맛보기 → 내 업무 적용"까지. 핵심은 **오전에 반드시 '됐다!' 경험**을 만드는 것.

## 학습 목표
- AX·바이브 코딩 이해 / 잘 시키는 법 / **동작하는 미니 도구 1개 완성** / 하네스·스킬 맛보기 / 적용 아이디어.

## 준비물 (강사)
- 시연 화면, 랩 [완성본](./lab-mini-tool.md), [슬라이드](./slides.md), 셋업 미완료자용 보조.

## 강의 노트 (핵심 메시지)
- **AX·바이브 코딩**: 코드를 짜는 게 아니라 "무엇을·왜"를 설명하면 AI가 만든다. 나=운전대(의도·검증), AI=엔진. (용어: [glossary](../_shared/glossary.md))
- **잘 시키는 법 4요소**: 무엇을 / 왜·맥락 / 예시·입력 / 제약. 안 되면 "더 단순하게 다시".
- **하네스 맛보기**: AI가 스스로 파일을 만들고·실행·수정하는 환경(Cursor). 변경은 **수락(Apply)** 후 실행해 확인.
- **Agent Skills 맛보기**: 반복 업무 절차를 `SKILL.md`로 적어두면 매번 같은 품질. description("언제 쓰나")이 핵심.
- **안전**: 개인정보 금지·사람이 검증([safety](../_shared/safety-ethics.md)).

## 실습
- **랩 A (90분)**: [lab-mini-tool](./lab-mini-tool.md) — 할 일 체크리스트 웹앱(또는 업무 계산기/요약기). 완성 기준=화면에서 동작+스크린샷.
- **랩 B (50분)**: 만든 도구에 기능 1개 추가(맥락 주고 단계적으로). 예) "새로고침해도 유지", "완료 개수 표시".

## 워크숍 (30분) — 내 업무 적용
- 각자 "내 업무 불편 1개 → AI로 만든다면 가장 작은 버전(MVP) 한 줄" 작성·공유.
- 강사: 너무 큰 아이디어는 **MVP로 축소**하도록 코칭.

## 강사 팁 / 흔한 막힘
- 셋업 미완료가 1순위 리스크 → OT~랩A 사이 조교 구제.
- "미리보기가 안 보여요" → HTML 한 파일로 만들고 브라우저로 열기, 수정 후 새로고침.
- 큰 걸 만들려는 사람 → "오늘은 가장 작은 한 가지만".
- 모범 멘트: *"완벽히 시키지 말고, 일단 시키고 결과 보고 고쳐봅시다."*
