# 직군 특화 예시 모음 (Industry / Role Examples) — 과정군 공용

> 강사가 학습자의 직군에 맞춰 **프로젝트 소재·예시 프롬프트**를 고를 수 있도록 정리했습니다.
> 비개발 트랙(N1/N3/N7)의 "내 업무 적용/팀 프로젝트"와, 개발 트랙(D1/D3/D7)의 "실전 워크플로우"에서 사용하세요.
> ⚠️ 모두 **샘플/가공 데이터**로 진행하고, 개인정보·기밀은 사용하지 않습니다([safety-ethics](./safety-ethics.md)).

---

## A. 비개발 직군별 (MVP 아이디어 + 예시 프롬프트)

### 마케팅
- 문제: 콘텐츠 카피·해시태그를 매번 수작업으로 변형.
- MVP: 제품 설명을 넣으면 채널별(인스타/블로그/메일) 카피 3종을 뽑는 한 페이지 도구.
- 프롬프트: `제품 설명을 붙여넣으면 인스타·블로그·메일용 카피 각 1개와 해시태그 5개를 보여주는 한 페이지 웹앱을 만들어줘. 한국어, 외부 전송 금지.`

### 인사/HR
- 문제: 면접 후기·평가 코멘트 정리에 시간 소요.
- MVP: 면접 메모를 넣으면 [강점/우려/추천여부] 요약 표로 정리.
- 스킬: `SKILL.md`로 "면접 평가 요약" 절차 고정(양식·금지사항: 차별적 표현 배제).

### 영업/세일즈
- 문제: 미팅 후 후속 메일·요약 작성 반복.
- MVP: 미팅 메모 → 고객용 후속 메일 초안 + 내부 요약 + 다음 액션.
- 프롬프트: `미팅 메모를 붙여넣으면 ① 고객 후속 메일(정중한 존댓말) ② 내부용 3줄 요약 ③ 다음 액션 목록을 만들어줘.`

### 운영/총무
- 문제: 여러 엑셀/시트의 정산·집계를 수작업 병합.
- MVP: 표 텍스트를 붙여넣으면 합쳐서 합계까지 내주는 도구(과정 샘플 `sample-data.csv` 활용).

### CS/고객지원
- 문제: 반복 문의 분류·답변 초안 작성.
- MVP: 문의 목록을 [환불/배송/기타]로 분류 + 유형별 답변 템플릿 제안.
- 스킬: "문의 답변" 스킬(톤·금지사항·에스컬레이션 규칙 포함).

### 재무/회계
- 문제: 비용 항목 분류·요약 리포트.
- MVP: 거래 내역(가공 CSV)을 항목별로 집계·요약하는 도구. ※ 숫자는 반드시 사람이 검증.

### 기획/PM
- 문제: 회의록·요구사항 정리, 일정 초안.
- MVP: 회의 메모 → 회의록 + 할 일(담당·기한) 표(과정의 `meeting-notes` 스킬 재사용).

### 교육/HRD
- 문제: 교육 자료·퀴즈 초안 제작.
- MVP: 주제를 넣으면 학습목표·요약·5문항 퀴즈를 생성하는 도구.

### 디자인
- 문제: 카피/마이크로카피·대체텍스트 작성.
- MVP: 화면 설명을 넣으면 버튼·안내문구·이미지 대체텍스트를 제안하는 도구.

> 사용법(강사): Day1 "내 업무 적용", Day4(N7)/Day3(N3) "팀 프로젝트 정의" 시 위 예시를 보기로 제시하되, **학습자 본인 문제를 우선**하도록 유도.

## B. 개발 직군/영역별 (실전 워크플로우 소재)

### 백엔드
- 버그 수정·리팩터·테스트 보강(과정 샘플 `discount.js` 또는 본인 API 레포).
- 스킬: `test-writer`, `commit-message`, `pr-describer`. 자동화: PR마다 테스트 CI.

### 프론트엔드
- 컴포넌트 리팩터, 접근성(a11y) 점검, 스토리/테스트 생성.
- 스킬: "컴포넌트 테스트 작성", "접근성 점검 체크리스트".

### 데이터/분석
- 데이터 정제 스크립트, 쿼리 작성·검증, 리포트 자동화.
- 주의: 실데이터 금지(가공/샘플), 결과 수치는 사람이 검증.

### DevOps/플랫폼
- CI 워크플로우 작성, 로그·이슈 분류, 런북 자동화.
- 자동화: 변경 시 린트·테스트 훅, PR 자동 점검.

### QA
- 테스트 케이스 생성, 엣지케이스 도출, 버그 리포트 표준화 스킬.

> 사용법(강사): D1 랩(버그·기능), D3/D7 랩(스킬·MCP·자동화)에 학습자 영역에 맞는 소재를 매핑. 비밀정보·토큰은 금지, 변경은 테스트·diff 리뷰로 검증.
