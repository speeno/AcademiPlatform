# 공통 용어집 (Glossary) — AX 교육 과정군 공용

> 모든 과정에서 동일하게 사용하는 표준 정의입니다. 슬라이드·운영안·실습에서 이 용어를 일관되게 씁니다.

- **AI / LLM(대규모 언어모델)**: 자연어로 지시하면 글·코드·계획을 생성하는 AI. 예) Claude.
- **AX (AI Transformation)**: 도구로서의 AI를 업무에 녹여 일하는 방식 자체를 바꾸는 것.
- **바이브 코딩(Vibe Coding)**: 코드를 직접 타이핑하지 않고, **자연어로 의도를 설명하면 AI가 코드를 만들고 고치는** 개발 방식. 사람은 "무엇을·왜", AI는 "어떻게"를 담당.
- **하네스(Harness)**: AI 에이전트가 실제로 **도구를 쓰고(파일 읽기·쓰기·실행) 결과를 보고 다음 행동을 정하는 실행 환경/루프**. 대표: Cursor, Claude Code(클로드의 코딩 도구). → "AI에게 손과 작업대를 쥐여주는 것".
  - 3요소: **도구(Tools)** = AI의 손 / **맥락(Context)** = AI의 눈·배경지식 / **루프(Loop)** = 행동→확인→다음행동 반복.
- **Agent Skills(스킬)**: AI가 특정 작업을 잘하도록 미리 정리해둔 **재사용 지식·절차 묶음**. 폴더 + `SKILL.md`로 구성. 평소엔 설명(description)만 알고 있다가, 관련 작업이 오면 본문 절차를 펼쳐 읽고 따른다(**점진적 노출**).
  - `SKILL.md` 구조: 상단 메타(`name`, `description`) + 본문(언제 쓰나 / 절차 / 출력 양식 / 규칙).
- **기술 파일(Technical Files)**: 프로젝트를 굴러가게 하고 AI에게 맥락을 주는 파일들 — README(설명서), 설정 파일, `SKILL.md`, 샘플 데이터 등.
- **MVP(최소 기능 제품)**: "가장 작은데도 쓸모 있는" 한 가지 버전. 범위를 줄이는 것이 핵심.
- **컨텍스트(맥락) 주기**: 예시·자료·규칙을 함께 제공해 AI 결과 품질을 높이는 것.
- **검증(Verification)**: AI 결과를 사람이 확인하는 것. 숫자·날짜·법규·계약은 반드시 사람이 대조.

> 개발 트랙 추가 용어(서브에이전트·MCP·컨텍스트 엔지니어링·프롬프트 캐싱 등)는 개발 과정 자료에서 별도 정의.
