# 에이전트 코딩 1Day
## 하네스 & Agent Skills로 개발 생산성 즉시 향상
개발자용

> "코드를 받는 게 아니라, **에이전트를 운용**한다."

---

## 오늘의 목표
- 에이전트 코딩 **멘탈 모델**
- 스펙→플랜→실행 **워크플로우**(버그·기능)
- 커스텀 **Agent Skill** + **CLAUDE.md**
- 컨텍스트·서브에이전트·검증

---

## 멘탈 모델 — 하네스
- 에이전트가 **도구를 쓰며** 일하는 환경(Claude Code · Cursor)
- 3요소: **도구**(파일·실행·검색) · **맥락** · **루프**(행동→확인→다음)
- 사람 = 목표·맥락 + **검증** / AI = 코드 작성

---

## 플랜-퍼스트
- 위험·큰 변경은 **계획부터** 받는다
```
목표: (한 문장). 영향 파일과 단계별 계획을 먼저. 코드는 아직 쓰지 말고.
```
- 검토·수정 후 "이 계획대로 진행"

---

## 효과적 프롬프팅
- **스펙**(무엇을·제약) → **플랜**(영향·단계) → **실행**(작게·diff 검증)
- 맥락: 관련 파일 명시(@), 오류 로그 그대로
- 규칙은 **CLAUDE.md / Rules** 에 고정

---

## 랩 1 — 실전 워크플로우
- 샘플 레포에서 **버그 수정** + **기능 추가**
- 버그: 증상·기대·실제(로그) → 최소 수정 + 회귀 테스트
- 변경 diff는 **반드시 읽고 수락**

---

## Agent Skills
- 반복 절차를 `SKILL.md`(`name`/`description`+절차)로
- **점진적 노출**: 필요할 때만 로드
- 팀 자산: 버전관리·공유

## CLAUDE.md (프로젝트 메모리)
- 컨벤션·테스트 방법·자주 쓰는 명령 고정 → 매 세션 자동 맥락

---

## 랩 2 — 스킬 & 메모리
- 우리 레포용 `SKILL.md` 1개(예: 커밋 메시지/테스트 작성)
- `CLAUDE.md`에 규칙·명령 정리 → 적용해 차이 체감

---

## 확장 & 안전
- **컨텍스트 관리**: 조사·검색은 서브에이전트에 위임 → 메인 맥락 절약
- **자동화**: 코드리뷰·테스트 생성, 훅/슬래시 커맨드
- **안전**: 비밀정보 금지 · 변경은 **테스트·실행·diff 리뷰** 후 머지

---

## 랩 3 (미니) & 마무리
- AI에게 **코드 리뷰**/**테스트 생성** 시키고 검증
- **산출물**: 적용 커밋 + `SKILL.md` + `CLAUDE.md`
