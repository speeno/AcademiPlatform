## 강의 노트
### ① 멘탈 모델
- **하네스**: 에이전트가 도구(파일·실행·검색)를 쓰며 **행동→결과 확인→다음 행동**을 반복하는 환경. Claude Code/Cursor가 그 환경.
- 사람의 역할: **목표·맥락 제공 + 변경 검증**. "코드 작성"은 위임, "판단·검증"은 우리.
- **플랜 모드/플랜-퍼스트**: 위험·큰 변경은 계획부터 받고 검토 후 실행.

### ② 효과적 프롬프팅
- 스펙(무엇을·제약) → 플랜(영향 파일·단계) → 실행(작게, diff 검증).
- 맥락: 관련 파일 명시, 오류 로그 그대로, 규칙은 `CLAUDE.md`. ([쿡북](../_shared/prompt-cookbook-dev.md))

### ③ Agent Skills & 프로젝트 메모리
- **Agent Skill**: 반복 작업 절차를 `SKILL.md`(`name`/`description` + 절차)로. 점진적 노출로 필요할 때만 로드. 팀 자산으로 버전관리·공유.
- **`CLAUDE.md`**: 프로젝트 규칙·컨벤션·자주 쓰는 명령을 고정 → 매 세션 자동 맥락.

### ④ 확장·안전
- **컨텍스트 관리**: 큰 작업은 서브에이전트에 조사 위임해 메인 맥락 절약.
- **자동화 기초**: 코드리뷰·테스트 생성 워크플로우, (Claude Code) 훅/슬래시 커맨드로 검증 자동화.
- **안전**: 비밀정보 금지, AI 변경은 **테스트·실행·diff 리뷰** 후 머지.

## 강사 팁 / 흔한 막힘
- "큰 변경을 한 번에 시킴" → 플랜-퍼스트 + 작게 쪼개기 유도.
- diff를 안 읽고 수락 → **반드시 리뷰 후 수락** 강조(사고 예방).
- 모델/도구 버전 질문 → **수업 시점 공식 문서** 기준으로 확인하도록 안내(자료에 버전 단정 금지).
- 샘플 레포가 없으면 본인 사이드 레포로(비밀정보 제거 확인).
