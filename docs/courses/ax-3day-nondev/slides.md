# Day 2
## Agent Skills & 기술 파일

## Agent Skills
- 반복 절차를 `SKILL.md`로 → **매번 같은 품질**
- 평소엔 설명만, 필요할 때 절차를 펼친다(점진적 노출)
- `description`("언제 쓰나")이 핵심

## 기술 파일
- README·설정·SKILL·샘플 데이터 = **AI에게 주는 맥락**

---

## 팀 구성
- 4~5인(직무 다양하게) + 팀 차터
- 가져온 불편 → **팀 문제 후보** → 1개로 좁히기

# Day 3
## 팀 프로젝트: 정의 → 빌드 → 데모

---

## 문제 정의 & MVP
- 한 문장 문제 + **측정 가능한 성공 기준**
- **MVP**: 가장 작은데도 쓸모 있는 한 가지 (범위 축소가 실력)
- 강사 게이트 통과 후 빌드

---

## 빌드 & 데모
- 분해표의 "한 줄 지시" = 빌드 대본
- 작게: 지시 → 수락 → 실행 → (오류면) 수정 → 다음
- 데모 6분: 문제 → 시연 → 성과 (백업 스크린샷 필수)

---

## 마무리

- **안전**: 개인정보 금지 · 사람이 검증
