# 강 30 — 2차 실기·주관식 모의고사 · 모범답안·채점기준

> AI Prompt Engineer · ISO/IEC 17024 · 2차 실기 대비
> 형식: 주관식 + 실기(프롬프트 작성) / 합격 **60점 이상**
> 과목 태그: [활용실제] [도구] [윤리] [이론]
> **이 문항은 플랫폼 문제은행(DB)에도 등록한다.**

---

## A. 주관식 (각 10점, 5문항 = 50점)

**1. [이론]** 트랜스포머가 RNN의 한계를 어떻게 극복했는지 2가지 이상 서술하시오.
**모범답안**: ① 셀프어텐션으로 거리와 무관하게 단어를 직접 연결해 **장기 의존성 문제**를 해결. ② 순차 처리 대신 **병렬 처리**가 가능해 대규모·고속 학습이 가능.
**채점**: 장기 의존성 해결(5) + 병렬 처리(5).

**2. [이론]** RLHF의 3단계를 순서대로 쓰고 각 단계를 한 줄로 설명하시오.
**모범답안**: ① **SFT**(지도 미세조정): 사람이 쓴 모범 답변으로 학습. ② **보상 모델 학습**: 답변 순위로 인간 선호 학습. ③ **강화학습**: 보상 모델 점수를 높이도록 최적화.
**채점**: 순서 정확(4) + 각 설명(2×3).

**3. [활용실제]** 좋은 프롬프트의 4대 요소를 쓰고 각각 예를 드시오.
**모범답안**: **역할**("너는 영양사다"), **맥락**("고혈압 환자 대상"), **예시**("이런 형식으로: …"), **제약**("200자, 표로"). 
**채점**: 4요소 명칭(4) + 예시(6).

**4. [도구]** 프롬프트 체이닝이 단일 프롬프트보다 유리한 경우와 이유를 서술하시오.
**모범답안**: 복잡·다단계 작업에서 유리. 작업을 분할해 **단계별 검증·수정**이 가능하고, 앞 단계 출력을 다음 입력으로 써 **품질과 통제력**이 향상되며 오류 추적이 쉽다.
**채점**: 분할/검증(5) + 통제·품질(5).

**5. [윤리]** RAG가 할루시네이션을 줄이는 원리와 한계를 서술하시오.
**모범답안**: 질문 관련 문서를 검색해 **근거로 함께 제공**하므로 모델이 추측 대신 자료 기반으로 답함 → 할루시네이션 감소, 출처 검증 가능. **한계**: 검색 자료가 부정확/부족하면 여전히 오류 가능, 0으로 만들지는 못함.
**채점**: 근거 제공 원리(6) + 한계(4).

## B. 실기 — 프롬프트 작성 (각 25점, 2문항 = 50점)

**6. [활용실제/도구]** 다음 요구로 **회의록 요약 봇**의 프롬프트를 작성하시오.
- 요구: 회의록 텍스트를 입력받아 ①핵심 요약 3줄 ②결정사항 ③담당자별 할 일 추출.
- 조건: 시스템 프롬프트 + 핵심 프롬프트(4요소) + 출력 형식 지정 포함.

**모범답안(예)**:
```
[시스템] 너는 회의록 정리 비서다. 사실만 사용하고 추측하지 않는다.
입력에 없는 내용은 "기재 없음"으로 표기한다.
[프롬프트] (역할) 회의록 분석가로서 (맥락) 아래 회의록을
(제약) 다음 형식의 표로 정리하라:
## 요약(3줄)
## 결정사항(목록)
## 할 일(담당자 | 업무 | 기한)
[회의록] """{내용}"""
```
**채점 루브릭(25)**: 시스템 프롬프트(5) · 4요소(8) · 출력 형식 지정(7) · 누락 처리/안전(5).

**7. [도구/윤리]** **사내 규정 기반 Q&A**(RAG) 프롬프트를 작성하시오.
- 조건: 제공 문서만 근거로 답하고, 없으면 "자료에 없음", 출처(문서·항목) 표기, 개인정보 요구 금지.

**모범답안(예)**:
```
[시스템] 너는 사내 규정 안내 도우미다. 아래 [자료]만 근거로 답하라.
자료에 없으면 "자료에 없음"이라 답하고 추측하지 마라.
답변 끝에 (출처: 문서명·항목)을 표기하라. 개인정보는 묻지 않는다.
[자료] """{검색된 규정}"""
[질문] {사용자 질문}
```
**채점 루브릭(25)**: 자료 한정(7) · 미존재 처리(6) · 출처 표기(6) · 윤리(개인정보)(6).

## 채점 기준 요약
- 주관식 50 + 실기 50 = **100점**, **60점 이상 합격**.
- 실기는 루브릭 항목별 부분점수 부여.
- 과목 태그별 득점 분포로 약점 진단.
