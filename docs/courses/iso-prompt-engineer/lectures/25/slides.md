# 강 25
## 페르소나·시스템 프롬프트 심화
AI Prompt Engineer · ISO/IEC 17024 — 2차 실기 대비

> "일관된 'AI 인격'을 운영하는 법."

---

## 학습 목표
1. **시스템 프롬프트** 의 역할을 안다
2. **일관된 페르소나** 운영법을 익힌다
3. 챗봇용 시스템 프롬프트를 작성한다

---

## 도입 — 매번 같은 캐릭터로
- 일반 프롬프트: 그때그때 지시
- 시스템 프롬프트: **대화 전체에 적용되는 규칙**
- 챗봇·서비스의 '기본 인격'을 정의

---

## 시스템 프롬프트란?
- 모델에 **항상 적용** 되는 최상위 지시문
- 역할·말투·금지사항·범위를 고정
- 사용자 메시지보다 **우선** 적용

---

## 구성 요소
1. **정체성**: "너는 큐미, 학습 도우미다"
2. **말투·톤**: "친근한 반말, 이모지 사용"
3. **범위**: "학습 관련만 답한다"
4. **금지**: "개인정보 요구 금지"

---

## 일관성 유지 전략
- 페르소나 속성을 **명확·구체** 로 고정
- 예외 상황 행동 규칙 명시
- 어긋나면 시스템 프롬프트로 **재정렬**

---

## 예시 — 학습 도우미 봇
```
너는 '큐미', 친근한 학습 도우미다.
- 말투: 따뜻한 존댓말, 이모지 1개
- 범위: 강의·학습 질문만
- 모르면: 모른다고 솔직히 답
- 금지: 시험 부정행위 도움
```

---

## 사용자 vs 시스템 프롬프트
| 구분 | 시스템 | 사용자 |
|---|---|---|
| 적용 | 전체 대화 | 해당 턴 |
| 내용 | 규칙·인격 | 개별 요청 |
| 우선순위 | 높음 | 낮음 |

---

## 실무 팁
- 너무 길면 핵심이 흐려짐 → **간결·우선순위**
- 충돌 규칙 피하기
- 배포 전 **다양한 입력으로 테스트**

---

## 정리 & 예상문제(실기)
- 시스템 프롬프트=전체 적용 규칙/인격
1. (주관식) 대화 전체에 적용되는 최상위 지시문은? → **시스템 프롬프트**
2. (실기) '상담 챗봇'용 시스템 프롬프트를 4요소로 작성하시오.
