# 강 15
## GPT vs 전통 검색 엔진
AI Prompt Engineer · ISO/IEC 17024 — 필기 대비

> "검색은 '찾아주고', GPT는 '만들어준다'."

---

## 학습 목표
1. **전통 검색** 의 작동 방식을 안다
2. **GPT** 와의 차이를 이해한다
3. 각각의 **장단점** 을 설명한다

---

## 도입 — 두 가지 답하는 방식
- 질문 "AI란 무엇인가?"
- 검색: 관련 **웹페이지 목록** 을 보여줌
- GPT: **하나의 정리된 답변** 을 생성

---

## 전통 검색 엔진
- **키워드 매칭** + **링크 순위**(예: 페이지랭크)
- 색인된 문서에서 관련 페이지를 찾아 나열
- 사용자가 직접 읽고 종합해야 함

---

## GPT (생성형 LLM)
- 질문의 **문맥을 이해** 하고 답을 **생성**
- 대화형, 후속 질문 가능
- 요약·번역·작성 등 가공된 결과 제공

---

## 핵심 차이 비교
| 구분 | 전통 검색 | GPT |
|---|---|---|
| 방식 | 키워드·링크 | 문맥 이해·생성 |
| 결과 | 페이지 목록 | 정리된 답변 |
| 최신성 | 실시간 색인 | 학습 시점 한계 |

---

## GPT의 장점·한계
- **장점**: 문맥 이해, 대화, 가공된 답
- **한계**: **할루시네이션**(없는 사실 생성)
- 학습 시점 이후 정보 모름 → 검증 필요

---

## 그래서 — 함께 쓴다
- 검색 결과를 LLM에 결합 = **RAG**
- 최신·정확한 근거 + 자연스러운 생성
- 강26에서 RAG 개념 학습

---

## 흔한 오해 (시험 주의)
- "GPT는 항상 최신 정보를 안다" → ❌ (학습 시점 한계)
- "검색은 답을 직접 생성한다" → ❌ (목록 제공)

---

## 정리 & 예상문제
- 검색=키워드·목록 / GPT=문맥이해·생성(할루시네이션 주의)
1. (4지선다) GPT의 대표적 한계는?
   ①키워드만 사용 ②**할루시네이션** ③링크 제공 ④색인 불가 → **②**
2. (O/X) 전통 검색은 결과로 페이지 목록을 제공한다. → **O**
