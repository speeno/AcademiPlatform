# 강 26 — 멀티모달·자료 결합(RAG) · 강의 스크립트

> AI Prompt Engineer · ISO/IEC 17024 · 1강 패키지 (강사용 운영안) · 2차 실기 대비

## 강의 개요
- **시간**: 약 30분 (슬라이드 11장 + 실습)
- **목표**: RAG 개념·흐름 + 근거 기반 프롬프트 작성
- **선수**: 강15·12 / **산출물**: 자료 결합 Q&A 프롬프트

## 시간 배분
| 구간 | 시간 | 슬라이드 |
|---|---|---|
| 도입·개념 | 7분 | 3~4 |
| 동작 흐름 | 6분 | 5~6 |
| 결합 프롬프트 | 8분 | 7~8 |
| 팁·실습·퀴즈 | 9분 | 9~11 |

## 도입 (멘트)
> "GPT는 모르면 지어냅니다(할루시네이션). 최신·사내 정보도 모르죠. 답을 근거 자료와 함께 주면 신뢰도가 확 올라갑니다. 이게 RAG이고, 우리 큐미 챗봇도 이 방식입니다."

## 전개 (슬라이드별 강의 포인트)
### S4 개념
- 검색+생성. 강15 RAG 예고 회수.
### S5~6 흐름
- 임베딩→벡터검색→맥락주입→생성. 4단계.
### S7~8 결합 프롬프트
- "자료만 근거로, 없으면 모른다고" 안전장치. 멀티모달 결합.

## 실습 (6분)
- 샘플 규정 문서로 근거 기반 Q&A 프롬프트 작성·실행.

## 정리 (멘트)
> "RAG는 근거를 줘서 헛소리를 줄입니다. 단, 0으로 만들진 못하니 검증은 필수. 다음 강은 프롬프트를 평가하고 개선하는 방법입니다."

## 예상문제 + 해설
1. **(주관식)** 외부 문서를 검색해 결합하는 기법? → **RAG**.
2. **(실기)** 근거 기반 Q&A 프롬프트(자료 한정·출처 요청) 작성 채점.
> 채점 루브릭: 근거 한정 지시·미존재 처리·출처 요청.
