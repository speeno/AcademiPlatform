# 강 23 — 프롬프트 설계 원리 · 강의 스크립트

> AI Prompt Engineer · ISO/IEC 17024 · 1강 패키지 (강사용 운영안) · 2차 실기 대비

## 강의 개요
- **시간**: 약 30분 (슬라이드 12장 + 실습)
- **목표**: 4요소(역할·맥락·예시·제약) 체득 + 설계 실습
- **선수**: 강16·17 / **산출물**: 4요소 프롬프트 1개

## 시간 배분
| 구간 | 시간 | 슬라이드 |
|---|---|---|
| 도입·4요소 | 8분 | 3~4 |
| 요소별 심화 | 10분 | 5~8 |
| 좋은예·체크 | 6분 | 9~10 |
| 실습·퀴즈 | 6분 | 11~12 |

## 도입 (멘트)
> "같은 AI라도 어떻게 묻느냐에 따라 답이 완전히 달라집니다. 프롬프트 엔지니어링은 '질문을 설계하는 기술'이고, 2차 실기의 핵심입니다."

## 전개 (슬라이드별 강의 포인트)
### S4 4요소
- 역할·맥락·예시·제약. **암기 필수**.
### S5~8 요소별
- 역할(페르소나), 맥락(대상·목적), 예시(few-shot), 제약(형식·길이·톤).
### S9 좋은 예
- 4요소를 모두 담은 완성형 프롬프트 시연.

## 실습 (6분)
- "회의록 요약" 작업을 4요소로 작성 → 실제 실행 → 결과 비교.

## 정리 (멘트)
> "역할·맥락·예시·제약 — 이 4가지를 챙기면 답의 품질이 달라집니다. 다음 강은 출력 형식 제어와 프롬프트 체이닝입니다."

## 예상문제 + 해설
1. **(주관식)** 프롬프트 4대 요소? → **역할·맥락·예시·제약**.
2. **(실기)** 임의 작업을 4요소로 작성 → 4요소 포함 여부로 채점.
> 채점 루브릭: 각 요소 1점(총 4점), 실행 결과 적절성 가점.
