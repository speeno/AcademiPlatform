# 강 29
## 캡스톤 — 업무용 프롬프트 설계
AI Prompt Engineer · ISO/IEC 17024 — 2차 실기 대비

> "배운 모든 것을 하나의 프로젝트로."

---

## 학습 목표
1. 실제 시나리오용 **프롬프트 세트** 를 설계한다
2. 4요소·체이닝·시스템 프롬프트를 **통합** 한다
3. 평가·개선까지 **완결** 한다

---

## 캡스톤 개요
- 실무 시나리오 1개 선택
- **프롬프트 세트 또는 미니 봇** 설계
- 설계 → 실행 → 평가 → 개선 → 발표

---

## 시나리오 예시
- 고객 문의 자동 응답 봇
- 회의록 → 요약·할일 추출 파이프라인
- 신입 교육용 Q&A 도우미
- 마케팅 카피 생성 세트

---

## 설계 단계 ① 정의
- 목적·사용자·성공 기준을 명확화
- "무엇을, 누구를 위해, 어떻게 좋은지"
- 평가 기준 미리 정의(강27)

---

## 설계 단계 ② 구성
- **시스템 프롬프트**(인격·규칙) 작성
- **4요소** 로 핵심 프롬프트 설계
- 필요 시 **체이닝** 으로 단계 분할

---

## 설계 단계 ③ 안전·근거
- **RAG** 로 근거 결합(필요 시)
- **윤리 체크**(편향·개인정보·검증)
- 한계·주의사항 명시

---

## 설계 단계 ④ 평가·개선
- 평가 기준으로 채점(A/B)
- 약점 수정 → 재실행 → 비교
- 최종본 **템플릿화**

---

## 산출물 형식
- 시나리오·목적서 1장
- 프롬프트 세트(시스템+핵심+체인)
- 평가표 + 개선 기록
- 데모 결과 캡처
