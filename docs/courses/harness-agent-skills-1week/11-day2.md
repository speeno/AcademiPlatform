# Day 2 — 하네스 이해 & 에이전트처럼 일 시키기 (강사 운영안)

> 오늘의 한 줄: **"AI에게 손과 작업대를 쥐여주면, 스스로 파일을 만들고·실행하고·고친다."**

## 학습 목표
- **하네스(harness)** 가 무엇인지(도구·맥락·반복 루프) 설명한다.
- Cursor의 Agent가 **여러 단계 작업**을 스스로 수행하는 과정을 관찰·통제한다.
- 맥락(예시·자료)을 주어 결과 품질을 끌어올린다.

## 준비물 (강사)

- "맥락 주기" 전후 비교 시연 자료

## 강의 노트
### ① 하네스란?
- 어제 Day1은 "한 번 시키고 결과 받기"였다면, 하네스는 **AI가 스스로 도구를 써가며 일하는 환경**.
- 하네스의 3요소:
  - **도구(Tools)**: 파일 읽기/쓰기/수정, 명령 실행, 검색 등 — AI의 "손".
  - **맥락(Context)**: 우리가 준 설명·예시·열린 파일들 — AI의 "눈과 배경지식".
  - **루프(Loop)**: 행동 → 결과 확인 → 다음 행동을 **스스로 반복**.
- 대표 하네스: **Cursor**(GUI, 비개발자 친화), **Claude의 코딩 도구**.
- 비유: Day1이 "심부름 한 번 시키기"라면, 하네스는 "작업대와 연장을 주고 **완성될 때까지 알아서 일하게** 하기". 단, **우리가 매 단계 확인·수정 지시**한다.
- 통제 포인트: 변경은 **수락(Apply)** 해야 적용된다 → 항상 무엇이 바뀌는지 보고 수락.

### ② 좋은 지시 vs 나쁜 지시 / 검증
- 나쁨: "잘 만들어줘" → 좋음: "F1을 만들어줘. 입력은 ~, 출력은 ~, 한국어로, 한 파일로."
- **작게 쪼개기**: 한 번에 하나의 기능. 다 되면 다음.
- **검증 습관**: 실제로 실행해 본다 / 예시 입력으로 결과 확인 / 숫자·날짜는 직접 대조.
- 막히면: "오류 메시지 전체를 붙이고, 원인을 설명한 뒤 고쳐줘."

## 실습
### 랩 2-1 (60분) — 다단계 작업 시키기
- 목표: 하네스가 **여러 파일·여러 단계**를 처리하는 걸 체험.
- 예시 지시(순차):
  ```
  1) mini-site 라는 폴더를 만들고, 그 안에 간단한 한국어 소개 웹페이지(index.html)를 만들어줘.
  2) 제목, 소개 문단, 연락 버튼을 넣어줘.
  3) 버튼을 누르면 "문의 접수됨" 알림이 뜨게 해줘.
  4) 색상을 파란색 테마로 다듬어줘.
  ```
- 각 단계에서 **무엇이 바뀌는지 확인 후 수락** → 브라우저로 열어 동작 확인.
- 일부러 오류 유도(예: "안 되는데?") → 오류를 붙여 **AI가 고치는 루프** 관찰.

### 랩 2-2 (80분) — 맥락 주고 품질 올리기
- "맥락 전/후" 비교: 같은 요청을 ① 그냥 ② 예시·규칙을 붙여서 → 결과 차이 체감.
- 각자 어제 미니 도구를 **한 단계 키우기**(기능 1개 추가) — 맥락(예시 데이터·원하는 형식)을 붙여 지시.

## 강사 팁 / 흔한 막힘
- "수락을 안 해서 안 바뀌어요" → 변경 적용(Apply/Accept) 다시 안내.
- AI가 너무 많이 한 번에 바꿔 헷갈림 → "**한 번에 한 가지만** 바꿔줘"로 통제.
- 팀 편성 시 직무 다양성 섞기(기획+운영+CS 등) → 문제 시야 넓어짐.
