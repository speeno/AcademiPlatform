# Day 1 — 시작: AI로 일하는 법 & 첫 성공 경험 (강사 운영안)

> 오늘의 한 줄: **"두려움을 없애고, 자연어로 시켰더니 진짜 동작하는 걸 직접 경험한다."**

## 학습 목표
- AX·LLM·바이브 코딩이 무엇인지 자기 말로 설명한다.
- AI에게 일을 시키는 기본기(명확한 지시)를 익힌다.
- 자연어 지시만으로 **동작하는 미니 도구 1개**를 완성한다.

## 준비물 (강사)
- 셋업 완료 현황(사전 점검) / 미완료자용 보조 데스크 30분
- 시연 화면(빔), 예시 프롬프트, [프롬프트 쿡북](./30-prompt-cookbook.md)
- 미니 결과물 완성본(백업 시연용)

## 강의 노트 (핵심 메시지)
### ① AX와 바이브 코딩
- **AX(AI Transformation)**: 도구로서의 AI를 업무에 녹여 일하는 방식 자체를 바꾸는 것.
- **바이브 코딩**: "코드를 짜는 것"이 아니라 **"무엇을 왜 만들지 설명하면 AI가 만들어 주는 것"**. 우리는 운전대(의도·검증)를 잡고, AI가 엔진(코드 작성).
- 비유: "요리를 직접 못 해도, 정확히 주문하면 셰프가 만들어 준다. 단, 맛은 내가 본다(검증)."
- 강조: **완벽한 지시보다 빠른 시도-확인-수정 반복**이 이긴다.

### ② 잘 시키는 법 (프롬프트 기본 4가지)
1. **무엇을** 원하는지 (결과물): "한 페이지짜리 체크리스트 웹페이지"
2. **왜/맥락**: "팀 회의 준비용", "한국어로"
3. **예시/입력**: 샘플 데이터·양식을 붙여준다
4. **제약**: "단순하게", "외부로 아무것도 보내지 마"
- 안 되면? → **"왜 안 됐는지 설명하고, 더 단순하게 다시"** 라고 요청.

## 실습
### 랩 1-1 (60분) — 환경 점검 & 첫 지시
- 모두 Cursor에서 `ax-week` 폴더 열기 → AI 패널에서:
  ```
  이 폴더에 notes.md 파일을 만들고, 오늘 배운 것을 적을 제목 3개(완료/배운점/질문)를 넣어줘. 한국어로.
  ```
- 생성 제안 **수락(Apply)** → 파일 확인. (셋업 미완료자는 이 시간에 해결)

### 랩 1-2 (90분) — 첫 미니 도구 (각자)
- 아래 중 하나를 골라 자연어로만 만들기(예시 프롬프트 제공):
  - **체크리스트 웹페이지**: "할 일을 입력하고 체크하면 줄이 그어지는 한 페이지 웹앱을 만들어줘. 파일 하나로, 한국어 UI로."
  - **문서 요약기**: "긴 글을 붙여넣으면 3줄 요약과 핵심 키워드를 보여주는 간단한 페이지" (또는 Claude로 직접 요약 실습)
  - **간단 계산기/환율·할인 계산 등 업무 계산기**
- 흐름: 만들기 → **실행/미리보기** → 안 되면 "오류 메시지째 붙여 다시" → 다듬기.
- 완료 기준: **화면에서 실제로 동작** + 스크린샷 저장.

## 강사 팁 / 흔한 막힘
- **셋업 미완료자**가 1순위 리스크 → 랩 1-1 시간에 조교가 전담.
- "미리보기가 안 떠요" → 결과물을 HTML 한 파일로 만들게 유도하고, 파일을 브라우저로 열게 안내.
- 너무 큰 걸 만들려는 사람 → "오늘은 **가장 작은 한 가지**만"으로 범위 축소.
- 잘 안 될 때 모범 멘트: *"AI에게 '방금 결과가 이렇게 안 됐어. 더 단순한 방법으로 다시 해줘'라고 말해보세요."*
