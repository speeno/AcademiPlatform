# 프롬프트 쿡북 — 비개발자용 Claude · Cursor 치트시트

> "코드를 몰라도, **잘 시키면** 됩니다." 아래 패턴을 그대로 복사해 쓰세요.

---

## 1. 모든 지시의 4요소 (기억할 한 가지)
> **무엇을 + 왜/맥락 + 예시/입력 + 제약**

```
[무엇을] 할 일을 입력하고 체크하면 줄이 그어지는 한 페이지 웹앱을 만들어줘.
[왜] 팀 회의 준비 체크용이야.
[예시] 항목 예: "자료 출력", "회의실 예약"
[제약] 파일 하나(HTML)로, 한국어 UI로, 외부로 아무것도 전송하지 마.
```

## 2. 작게 쪼개 시키기 (가장 중요한 습관)
한 번에 다 만들지 말고 한 단계씩:
```
1단계: 빈 페이지에 제목 "오늘의 할 일"과 입력칸, 추가 버튼만 만들어줘.
```
→ 동작 확인 후
```
2단계: 추가 버튼을 누르면 입력한 내용이 아래 목록에 추가되게 해줘.
```
→ 확인 후
```
3단계: 각 항목에 체크박스를 넣고, 체크하면 글자에 줄이 그어지게 해줘.
```

## 3. 오류·안 될 때 (복붙용)
```
방금 결과가 이렇게 안 됐어: (화면에 뜬 오류 메시지나 증상 그대로 붙여넣기)
왜 그런지 한 문장으로 설명하고, 더 단순한 방법으로 고쳐줘.
```
```
지금 화면에 아무것도 안 보여. 가장 단순하게 "글자 한 줄이라도 보이는" 상태로 되돌려줘.
```

## 4. 맥락(자료)을 줘서 품질 올리기
```
아래는 우리가 쓰는 보고서 예시야. 이 형식과 말투를 따라줘:
(좋은 예시 붙여넣기)
---
이제 아래 내용으로 같은 형식의 보고서를 만들어줘:
(이번 내용)
```

## 5. 스킬(SKILL.md) 호출하기
```
my-skills/meeting-notes/SKILL.md 의 절차에 따라, 아래 회의 메모를 정리해줘:
(회의 메모 붙여넣기)
```
> 결과가 절차를 안 따르면: "SKILL.md의 절차/규칙을 반드시 지켜서 다시 해줘."

## 6. 결과 검증 요청
```
방금 결과에서 숫자와 날짜가 입력과 일치하는지 표로 비교해서 보여줘.
지어낸 내용이 있으면 표시해줘.
```

## 7. 자주 쓰는 업무 프롬프트 템플릿
- **요약**: `아래 글을 핵심 3줄 + 키워드 5개로 요약해줘. 한국어로.`
- **표 정리**: `아래 텍스트를 [이름, 부서, 요청사항] 3열 표로 정리해줘.`
- **초안 작성**: `아래 조건으로 안내 메일 초안을 써줘. 정중한 존댓말, 5문장 이내.`
- **분류**: `아래 문의들을 [환불/배송/기타]로 분류해 표로 보여줘.`
- **아이디어→계획**: `이 아이디어를 일주일 안에 만들 수 있는 가장 작은 버전(MVP)으로 제안해줘.`

## 8. Cursor 사용 팁 (하네스)
- 시킬 땐 **관련 파일을 먼저 열어두면** AI가 맥락을 더 잘 잡습니다.
- AI가 제안한 변경은 **무엇이 바뀌는지 보고 수락(Apply/Accept)**. 한 번에 너무 많이 바꾸면 "한 가지만 바꿔줘".
- "현재 폴더에", "이 파일에"처럼 **대상 위치를 명시**하면 엉뚱한 곳에 안 만듭니다.

## 9. Claude(채팅) 사용 팁
- 기획·문구·요약·분류 등 **생각 정리/글쓰기**에 강합니다.
- 긴 자료는 붙여넣고 "이 안에서만 근거해서 답해줘"라고 하면 헛소리(지어내기)를 줄입니다.
- 모르면 "모르면 모른다고 해줘"를 덧붙이세요.
