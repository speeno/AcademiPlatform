# Day 3
## Agent Skills & 기술 파일 만들기

> "한 번 잘 적어두면, 매번 똑같이 잘하는 우리 팀의 AI 자산."

---

## 오늘의 목표
- **Agent Skills**(`SKILL.md`)의 개념·구조를 안다
- 업무 절차 1개를 **동작하는 SKILL.md**로 만든다
- 프로젝트 **기술 파일 세트**를 만든다

---

## Agent Skills란?
- AI가 특정 작업을 잘하도록 **미리 정리한 지식·절차 묶음**
- 폴더 + **`SKILL.md`** 파일로 구성
- 비개발자도 **글로** 작성 가능 📝

---

## 어떻게 작동하나 (점진적 노출)
- 평소엔 **제목·설명(description)만** 알고 있다가
- 관련 요청이 오면 **본문 절차를 펼쳐 읽고** 따른다
- → 그래서 **description("언제 쓰나")** 가 가장 중요

---

## 왜 좋은가
- ♻️ **재사용**: 매번 다시 설명 안 함
- 📏 **일관성**: 항상 같은 품질·양식
- 👥 **공유**: 팀 자산

비유: 신입에게 주는 **업무 매뉴얼**

---

## SKILL.md 구조
```
---
name: meeting-notes
description: 회의 메모를 받아 회의록·할 일을 정리할 때 사용
---
# 회의록 정리 스킬
## 언제 쓰나 / ## 절차 / ## 출력 양식 / ## 규칙
```
- 상단: 메타(언제 쓰나)
- 본문: 절차·양식·규칙

---

## 기술 파일(Technical Files)
- 📄 **README**: 무엇을·왜·어떻게 실행
- ⚙️ **설정 파일**: 제목·옵션
- 🧩 **SKILL.md**: 반복 절차
- 🗂️ **샘플 데이터**: 입력 예시(가공)

→ 이 파일들이 곧 **AI에게 주는 맥락**

---

## 실습
- **랩 3-1** 회의록 `SKILL.md` 만들고 호출 → 안 맞으면 **스킬을 고쳐** 재시도
- **랩 3-2** 프로젝트 기술 파일 세트(README·config·sample-data·SKILL)

---
