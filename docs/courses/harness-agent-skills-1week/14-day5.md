# Day 5 — 빌드 완료 & 데모 & 이어가기 (강사 운영안)

> 오늘의 한 줄: **"동작하는 결과물을 완성해 시연하고, 회사로 가져가 이어갈 길을 만든다."**

## 학습 목표
- 결과물을 마무리하고 **검증 체크리스트**로 점검한다.
- 5~7분 **데모**로 문제·해결·성과를 전달한다.
- 한계·안전을 이해하고 **사내 확산 다음 단계**를 계획한다.

## 준비물 (강사)
- 검증 체크리스트, [데모 대본](./templates/demo-script.md), [평가 루브릭](./40-assessment-rubric.md)
- 데모 타이머, 발표 순서, (선택) 수료증 발급 준비

## 강의 노트
### 마무리 검증 체크리스트
- [ ] 핵심 기능(MVP)이 **실제로 동작**하는가 (샘플 입력으로 시연)
- [ ] 예외 입력(빈 값/이상한 값)에서 크게 깨지지 않는가
- [ ] 결과(숫자·날짜·문구)를 **사람이 한 번 검증**했는가
- [ ] 개인정보·기밀이 들어가 있지 않은가
- [ ] 처음 보는 사람이 **30초 안에** 무엇인지 이해하는가

### 한계 & 안전
- AI 결과는 **틀릴 수 있다** → 중요한 판단·수치·법규는 반드시 사람이 확인.
- **개인정보·기밀** 입력 금지. 외부 전송 여부 점검.
- 만든 도구의 **적용 범위**를 분명히(어디까지 믿을 수 있는지).

### 사내 확산 다음 단계
- 만든 **파일·`SKILL.md`** 를 그대로 가져가 동료와 공유.
- "다음에 키울 기능 3가지" + "누가 이어받나" 정하기.
- 사내 보안/IT와 협의가 필요한 부분 식별.

## 실습 / 데모
### 랩 5-1 (105분) — 빌드 마무리
- 남은 태스크 완성 → 기능 연결 → 검증 체크리스트로 점검.
- **데모 시나리오에 맞춘 시연 데이터** 준비(라이브 실패 대비 스크린샷/녹화도).

### 데모 데이 (90분)
- 팀별 6분 발표([데모 대본](./templates/demo-script.md) 순서) + 2분 Q&A.
- 모든 팀원이 한 마디씩. 라이브 시연 우선, 실패 시 백업 자료.

## 산출물 (수료 핵심)
- **동작하는 팀 결과물** + **데모 발표** + **다음 단계 계획서**.

## 수료 / 평가
- [평가 루브릭](./40-assessment-rubric.md)에 따라 채점 → 출석·산출물·데모 충족 시 수료증 발급.

## 강사 팁 / 흔한 막힘
- 마감 압박으로 범위 폭증 → "오늘은 **완성·시연 가능**이 최우선. 새 기능 금지."
- 라이브 데모 실패가 가장 흔함 → **반드시 백업(스크린샷/녹화)** 준비 강제.
- 발표 울렁증 → 대본 빈칸을 미리 채우고 1회 리허설로 충분.
- 회고는 "오늘"이 아니라 "**일주일 전체에서 가장 크게 바뀐 생각**"을 묻기 → 학습 정착.
