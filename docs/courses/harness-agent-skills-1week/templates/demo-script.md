# 데모 대본 (Demo Script) — 5~7분

> 마지막 날 발표용. 빈칸을 채우고, 한 번 리허설하면 충분합니다.
> 좋은 데모 = "문제에 공감 → 해결을 직접 보여줌 → 배운 점". 슬라이드보다 **실제 동작 시연**이 핵심.

---

**팀명**: ____________   **발표자**: ____________   **시간 배분(총 6분)**

### ① 오프닝 — 문제 (1분)
- "저희는 ____________(누가) 가 ____________(무엇이) 불편한 문제를 풀었습니다."
- 현재는 이렇게 합니다(현 상황 한 문장): ____________
- 그래서 이런 비용이 듭니다(시간/실수 등): ____________

### ② 우리가 만든 것 (30초)
- 한 문장 소개: "저희는 ____________ 를 만들었습니다."
- 입력 → 결과: ____________ 를 넣으면 ____________ 가 나옵니다.

### ③ 라이브 시연 (2~3분) ★핵심
> 말로 설명하지 말고 **직접 보여주기**. 미리 시연 데이터 준비.
- 시연 순서:
  1. ____________ (예: 샘플 파일 넣기)
  2. ____________ (예: 버튼 클릭 / AI에 요청)
  3. ____________ (결과 보여주기)
- 백업: 라이브가 실패할 때 보여줄 **스크린샷/녹화** 준비 ✅

### ④ 어떻게 만들었나 (1분)
- 핵심 도구: Cursor/Claude로 바이브 코딩
- 인상 깊었던 순간 1개: ____________
- 사용한 스킬/기술 파일: ____________

### ⑤ 성과 & 다음 단계 (1분)
- 효과(성공 기준 대비): 예) 2시간 → 5분
- 한계/아직 안 된 것: ____________
- 회사에서 이어가려면: ____________

### ⑥ 마무리 (15초)
- "감사합니다. 질문 받겠습니다."

- [ ] 시연 데이터·환경 미리 세팅
- [ ] 실패 대비 스크린샷/녹화 준비
- [ ] 6분 안에 끝나는지 1회 리허설
- [ ] 모든 팀원이 한 마디씩(역할 분담)
