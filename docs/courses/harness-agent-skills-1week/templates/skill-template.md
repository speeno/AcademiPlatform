# SKILL.md 작성 템플릿 & 가이드

> **Agent Skill** 이란? AI가 특정 작업을 잘하도록 미리 정리해 둔 **재사용 지식·절차 묶음**입니다.
> 폴더 하나 + 그 안의 `SKILL.md` 파일로 만듭니다. AI는 평소엔 제목/설명만 알고 있다가, **관련 작업이 오면 그 안의 자세한 절차를 펼쳐 읽고** 그대로 따라 합니다(점진적 노출).
> 비개발자도 **글로** 작성할 수 있습니다.

---

## 1. 폴더 구조 (예)
```
my-skills/
└─ weekly-report/
   └─ SKILL.md        ← 이 파일이 핵심
   └─ (선택) 예시.txt, 양식.md 등 참고 파일
```

## 2. SKILL.md 기본 형식
파일 맨 위 `---` 사이는 **설명표(메타)**, 그 아래는 **실제 절차**입니다.

```markdown
---
name: weekly-report
description: 주간 업무 보고서를 정해진 양식으로 작성할 때 사용. 팀원들의 한 주 내용을 받아 요약·정리한다.
---

# 주간 보고서 작성 스킬

## 언제 쓰나
사용자가 "주간보고", "위클리", "이번 주 보고서" 등을 요청할 때.

## 따라야 할 절차
1. 입력에서 [완료한 일 / 진행 중 / 다음 주 계획 / 이슈] 4가지를 뽑는다.
2. 아래 양식으로 정리한다.
3. 문장은 짧고 명확하게, 존댓말로 쓴다.

## 출력 양식
- ## 이번 주 완료
- ## 진행 중
- ## 다음 주 계획
- ## 이슈/도움 필요

## 규칙
- 추측해서 내용을 지어내지 않는다(입력에 없으면 "내용 없음").
- 숫자·일정은 입력 그대로 사용한다.
```

## 3. 좋은 스킬을 쓰는 요령 (비개발자용)
- **description이 가장 중요**: "언제 이 스킬을 써야 하는지"를 한 문장으로. AI는 이걸 보고 불러올지 결정합니다.
- **절차는 단계로**: 사람이 신입에게 알려주듯 1, 2, 3 순서로.
- **규칙·금지사항 명시**: "지어내지 마라", "한국어로", "개인정보 요구 금지" 등.
- **예시를 곁들이면 품질↑**: 좋은 결과물 예시를 함께 넣기.
- **짧고 구체적으로**: 너무 길면 오히려 안 따릅니다. 핵심만.

## 4. 우리 팀 스킬 만들기 (빈칸 채우기)
```markdown
---
name: ____________________            (영문 소문자-하이픈, 예: meeting-notes)
description: ____________________      (언제 쓰는지 한 문장)
---

# (스킬 제목)

## 언제 쓰나
____________________

## 따라야 할 절차
1. ____________________
2. ____________________
3. ____________________

## 출력 양식
____________________

## 규칙
- ____________________
```

## 5. 만든 스킬 사용해 보기 (Cursor/Claude)
1. 위 내용을 프로젝트 안 `my-skills/<이름>/SKILL.md` 로 저장.
2. AI 패널에 작업을 요청하며 스킬을 참조하게 한다. 예:
   ```
   my-skills/weekly-report/SKILL.md 의 절차에 따라, 아래 내용으로 주간보고서를 만들어줘:
   (여기에 이번 주 내용 붙여넣기)
   ```
3. 결과가 절차/양식을 따르는지 확인하고, 안 맞으면 SKILL.md의 절차·규칙을 다듬어 다시 시도.

> 핵심: 스킬은 **한 번 잘 써두면 매번 똑같이 잘하는** 우리 팀의 자산이 됩니다.
