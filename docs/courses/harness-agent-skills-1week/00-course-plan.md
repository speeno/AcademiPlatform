## 2. 학습 성과 (Learning Outcomes)
수료 시 학습자는 다음을 할 수 있다.
1. **개념 이해**: LLM·에이전트·**하네스(harness)**·**Agent Skills**·**바이브 코딩**이 무엇이며 왜 비개발자에게 강력한지 설명한다.
2. **문제 정의**: 막연한 업무 불편을 "AI로 풀 수 있는 문제"로 구체화하고 범위를 정한다.
3. **기술·업무 분해**: 목표를 달성하기 위한 필요 기능·데이터·단계(태스크)를 정의한다.
4. **기술 파일 작성**: 프로젝트 구조, 설정 파일, **SKILL.md** 등 재사용 가능한 기술 파일을 만든다.
5. **바이브 코딩 실행**: Claude/Cursor에게 의도를 전달하고, 생성→실행→검증→반복으로 결과물을 완성한다.
6. **협업·전달**: 팀으로 결과물을 빌드하고, 데모하고, 이후 회사에서 이어갈 방법을 안다.

## 3. 핵심 개념 정의 (강의 표준 용어)
교육 자료 전반에서 아래 정의를 일관되게 사용한다(슬라이드·실습서 공통).

- **LLM(대규모 언어모델)**: 자연어로 지시하면 글·코드·계획을 생성하는 AI. 예) Claude.
- **바이브 코딩(Vibe Coding)**: 코드를 직접 타이핑하지 않고, **자연어로 의도를 설명하면 AI가 코드를 만들고 고치는** 개발 방식. 사람은 "무엇을/왜"에 집중하고 AI가 "어떻게"를 담당.
- **하네스(Harness)**: AI 에이전트가 실제로 **도구를 쓰고(파일 읽기·쓰기·명령 실행), 결과를 보고, 다음 행동을 정하는 실행 환경/루프**. Cursor와 Claude Code(클로드의 코딩 도구)가 대표적인 하네스. → "AI에게 손과 작업대를 쥐여주는 것".
- **Agent Skills(스킬)**: AI가 특정 작업을 잘하도록 미리 정리해둔 **재사용 지식·절차 묶음**. 폴더 안의 `SKILL.md`(설명 + 절차)로 구성되며, 필요할 때만 불러와 쓴다(점진적 노출). 비개발자도 글로 작성 가능.
- **기술 파일(Technical Files)**: 프로젝트를 굴러가게 하는 파일들 — 프로젝트 설명서, 설정 파일, `SKILL.md`, 데이터 샘플 등.
- **문제 정의서 / 기술·업무 분해표**: 만들기 전에 "무엇을 왜 만들고, 무엇이 필요한지"를 적는 양식(본 과정 제공 템플릿).

> ⚠️ 제작 시 정확성 기준: 하네스/Agent Skills/Claude 관련 서술은 최신 공식 개념(SKILL.md 구조, 점진적 노출, 도구 사용 루프)에 맞춰 검수한다. 모델명·가격 등 변동 정보는 단정하지 않고 "수업 시점 최신 확인" 박스로 처리.

## 4. 5일 커리큘럼 (일자별 설계)

각 일자 공통 구성: ① 학습목표 ② 타임테이블 ③ 핵심개념(강의) ④ 실습/랩 ⑤ 산출물 ⑥ 과제(다음 날 준비).

### Day 1 — 시작: AI로 일하는 법 & 첫 성공 경험
- **목표**: 두려움 제거 + 도구 셋업 + "자연어로 시켰더니 진짜 됐다"는 첫 성공.
- 핵심개념: AX란 무엇인가 / LLM·바이브 코딩 개요 / 잘 시키는 법(프롬프트 기본).
- 실습:
  - (랩 1-1) 환경 셋업 점검(Claude·Cursor 로그인, 프로젝트 폴더 만들기).
  - (랩 1-2) 첫 바이브 코딩: "내 업무용 간단 도구" 한 개 만들어 화면에 띄우기(예: 간단 계산기/체크리스트/문서 요약기).
- 산출물: 동작하는 미니 결과물 1개 + 스크린샷.
- 과제: 본인 업무 불편 3가지 적어오기(프로젝트 후보).

### Day 2 — 하네스 이해 & 에이전트처럼 일 시키기
- **목표**: 하네스(Cursor/Claude Code)가 어떻게 "스스로 파일을 만들고 고치고 실행"하는지 체득.
- 핵심개념: 하네스의 작동(도구·컨텍스트·반복 루프) / 좋은 지시 vs 나쁜 지시 / 결과 검증하는 법.
- 실습:
  - (랩 2-1) 하네스에게 여러 단계 작업 시키기(폴더 구조 생성 → 파일 작성 → 실행 → 오류 수정 반복).
  - (랩 2-2) "맥락 주기": 예시·자료를 붙여 결과 품질을 끌어올리기.
- 산출물: 2~3개 파일로 구성된 작은 프로젝트.
- 과제: 후보 문제 1개 골라 "왜 중요한지 5문장" 작성.

### Day 3 — Agent Skills & 기술 파일 만들기
- **목표**: 재사용 가능한 나만의/팀의 **스킬(SKILL.md)** 과 기술 파일을 작성.
- 핵심개념: Agent Skills 개념·구조(`SKILL.md`, 언제 불러오나) / 기술 파일 종류 / 좋은 스킬 작성법.
- 실습:
  - (랩 3-1) 업무 절차 1개를 `SKILL.md`로 작성하고 하네스에서 실제로 호출.
  - (랩 3-2) 프로젝트 기술 파일 세트 만들기(프로젝트 설명·설정·샘플 데이터).
- 산출물: 동작하는 `SKILL.md` 1개 + 기술 파일 세트.
- 과제: 팀 편성 확인, 팀 문제 후보 합치기.

### Day 4 — 문제 정의 & 팀 프로젝트 빌드 (1) 
- **목표**: 팀별 문제를 정의하고 기술·업무로 분해 후, 본격 빌드 착수.
- 핵심개념: 문제 정의 캔버스 / 범위 정하기(MVP) / 기술·업무 분해표 / 역할 분담.
- 실습:
  - (랩 4-1) **문제 정의서** 작성(템플릿) → 강사 피드백 게이트.
  - (랩 4-2) **기술·업무 분해표** 작성 → 빌드 시작(바이브 코딩 스프린트).
- 산출물: 승인된 문제 정의서 + 분해표 + 빌드 중간본.
- 과제: 데모 시나리오 초안.

### Day 5 — 빌드 완료 & 데모 & 이어가기
- **목표**: 결과물 완성 → 데모 → 회사에서 지속하는 법 학습.
- 핵심개념: 마무리·검증 체크리스트 / 데모하는 법 / 한계와 안전(개인정보·검증) / 다음 단계(사내 확산).
- 실습:
  - (랩 5-1) 빌드 마무리 스프린트 + 검증.
  - (랩 5-2) 팀별 5~7분 데모 + 상호 피드백 + 회고.
- 산출물: **동작하는 팀 결과물** + 데모 자료 + 회고/다음 단계 계획.
- 수료: 평가 루브릭 충족 시 수료(자격/배지 연계 가능).

## 5. 팀 프로젝트 트랙 (전 과정 관통)
문제 → 정의 → 분해 → 빌드 → 데모의 흐름을 Day1~5에 분산.

1. **발굴**(D1~D2): 업무 불편 수집.
2. **정의**(D4): 문제 정의서로 "무엇을 왜" 확정 + 강사 게이트.
3. **분해**(D4): 기술·업무 분해표로 "무엇이 필요한지/어떤 단계로" 확정.
4. **빌드**(D4~D5): 바이브 코딩 스프린트(Claude/Cursor).
5. **데모·전달**(D5): 시연 + 사내 확산 계획.

권장 프로젝트 예시(비개발자 친화): 반복 문서 자동 생성기, 사내 FAQ/가이드 챗봇, 데이터 정리·리포트 자동화, 간단 신청/체크 웹폼, 회의록 요약·할 일 추출 도구 등.
