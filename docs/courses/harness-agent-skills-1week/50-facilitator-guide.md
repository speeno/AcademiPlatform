# 강사 가이드 (Facilitator Guide)

> 이 과정을 처음 맡는 강사도 안정적으로 운영할 수 있도록, **운영 원칙·시간배분·흔한 막힘·즉답·안전**을 정리했습니다.

---

## 1. 운영 철학 (5원칙)
1. **작게, 자주**: 큰 강의보다 짧은 설명 + 즉시 실습.
2. **빠른 성공**: Day1 안에 "됐다!" 경험을 반드시 만든다(이탈 방지).
3. **막히면 빨리 도움**: "혼자 30분 헤매기" 금지 → 팀→조교→강사.
4. **범위를 줄여라**: 비개발자 프로젝트 실패의 90%는 과욕. MVP를 사수.
5. **사람이 검증**: AI 결과는 항상 사람이 확인하도록 습관화.

## 2. 사전 준비 (개강 D-3 ~ D-1)
- 학습자 **셋업 가이드** 발송 + 사전 완료 독려(미완료자가 첫날 최대 리스크).
- **방화벽 점검**: `claude.ai`, `cursor.com` 접근 가능 여부 사내 확인.
- 계정/비용 정책 확정(유료 플랜 필요 여부).
- 강의장: 빔/대형화면, 콘센트, 좌석은 **팀 단위 섬 배치**.
- 백업: 각 랩 **완성본**(`labs/`)을 강사 PC에 준비(시연·구제용).

## 3. 일자별 시간배분 & 리스크
| 일자 | 시간 핵심 | 최대 리스크 | 대응 |
|---|---|---|---|
| Day 1 | 랩 1-2(90분) 사수 | 셋업 미완료 | 조교가 랩1-1 시간에 전담 구제 |
| Day 2 | 하네스 루프 체감 | "수락 안 해서 안 바뀜" | 변경 Apply 반복 안내 |
| Day 3 | description 잘 쓰기 | 스킬이 안 불려옴 | "언제 쓰나"를 구체화 |
| Day 4 | **게이트로 범위 통제** | 범위 과욕 | MVP로 가차없이 축소 |
| Day 5 | 완성·시연 우선 | 라이브 데모 실패 | 스크린샷/녹화 백업 강제 |

## 4. 흔한 막힘 Top & 즉답
- **"미리보기가 안 보여요"** → HTML 한 파일로 만들게 하고 브라우저로 직접 열기. 수정 후 새로고침(F5).
- **"AI가 엉뚱한 데 파일을 만들어요"** → 지시에 "이 폴더에/이 파일에"처럼 **위치 명시**.
- **"한 번에 너무 많이 바뀌어 헷갈려요"** → "한 가지만 바꿔줘"로 통제.
- **"오류가 났어요"** → 오류 메시지를 **그대로 붙여** "원인 설명하고 고쳐줘".
- **"AI가 내용을 지어내요"** → "주어진 자료 안에서만, 없으면 '내용 없음'"이라고 규칙 추가.
- **"스킬이 안 따라져요"** → 호출 시 **경로 명시** + 규칙에 "이 절차를 반드시 따르라".

## 5. 진도 격차 관리
- **빠른 팀**: "기능 1개 더", "새로고침해도 저장되게", "동료에게 가르쳐 주기"(피어 티칭).
- **느린 팀**: 강사·조교 밀착, 완성본 일부 제공, 범위 추가 축소.
- 데일리 체크인(10분)으로 매일 막힌 점을 조기에 파악.

## 6. 안전·윤리 (강사가 지킬 것)
- 실데이터 사용 시도 즉시 **샘플/가공본**으로 전환.
- 개인정보·기밀·비밀번호·API 키 입력 발견 시 즉시 중단·삭제 안내.
- 결과의 한계 강조: 숫자·법규·계약은 **반드시 사람이 검증**.

## 7. 자주 받는 질문(FAQ) 답변
- **"코딩 1도 모르는데 따라갈 수 있나요?"** → 네. 자연어로 시키는 게 전부입니다. Day1에서 바로 성공합니다.
- **"회사에서 계속 쓸 수 있나요?"** → 네. 파일·스킬을 그대로 가져갑니다. Day5에서 확산 방법을 다룹니다.
- **"제 업무 데이터를 써도 되나요?"** → 가공/익명 데이터만. 민감정보 금지.
- **"Cursor와 Claude 중 뭘 쓰나요?"** → Cursor를 메인(만들기), Claude를 보조(기획·문구·검토)로 함께.

## 8. 에스컬레이션
1. 팀 내 5분 시도 → 2. 옆 팀/조교 → 3. 강사.
- 강사도 막히면: 완성본 비교 → 더 작은 단위로 재시도 → 안 되면 백업 완성본으로 진행.

## 9. 강사 멘트 모음 (복붙용)
- *"완벽하게 시키려 하지 말고, 일단 시키고 결과를 보고 고쳐봅시다."*
- *"지금 되는 가장 작은 것부터 보여주세요."*
- *"오류 메시지를 그대로 AI에게 붙여 보세요. '왜 그런지 설명하고 고쳐줘'."*
- *"이번 주 목표는 완벽이 아니라 **동작하는 것 하나**입니다."*

## 10. 과정 후
- 만족도/난이도 설문(5분), 결과물 회수(동의 시 사례로 활용).
- 우수 결과물은 사내 확산 사례로 공유.
- 수료 판정([루브릭](./40-assessment-rubric.md)) → 수료증 발급.
- 후속 제안: 3일/1일 심화, 팀별 코칭, 사내 AX 표준화.
