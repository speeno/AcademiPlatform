# 랩 3-1 — 동작하는 SKILL.md 만들기: 회의록 스킬 (Day 3)

> 목표: 반복 업무 절차 1개를 **재사용 가능한 `SKILL.md`** 로 만들고, 하네스에서 호출해 **매번 같은 품질**의 결과를 얻는다.
> 결과물: `my-skills/meeting-notes/SKILL.md`

## 진행 단계
**1단계 — 파일 만들기**
```
my-skills/meeting-notes/SKILL.md 파일을 만들어줘. 회의 메모를 받아 회의록과 할 일을 정리하는 스킬이야.
```
**2단계 — 내용 채우기** (아래 완성본을 참고해 절차·양식·규칙을 작성)

**3단계 — 호출해서 써보기**
```
my-skills/meeting-notes/SKILL.md 의 절차에 따라, 아래 회의 메모를 정리해줘:
(아래 "예시 입력" 붙여넣기)
```
**4단계 — 다듬기**
- 결과가 양식/규칙을 안 따르면, **SKILL.md를 고쳐서** 다시 호출(← 이 반복이 핵심 학습).

## 확인(검증)
- [ ] 결과가 "회의 요약 / 결정 사항 / 할 일(표)" 양식을 따른다
- [ ] 할 일이 "담당자-할 일-기한"으로 나온다
- [ ] 입력에 없는 내용을 지어내지 않았다

---

## ✅ 완성본 (정답) — `my-skills/meeting-notes/SKILL.md`
```markdown
---
name: meeting-notes
description: 회의 메모·녹취·대화를 받아 회의록과 할 일(액션 아이템)을 정리할 때 사용. "회의록", "정리해줘", "할 일 뽑아줘" 같은 요청에 사용한다.
---

# 회의록 정리 스킬

## 언제 쓰나
사용자가 회의 메모/대화를 주고 정리·요약·할 일 추출을 요청할 때.

## 따라야 할 절차
1. 입력에서 [결정 사항 / 논의 내용 / 할 일] 을 구분해 뽑는다.
2. 할 일은 "담당자 - 할 일 - 기한" 으로 정리한다. 입력에 없으면 "미정"으로 둔다.
3. 아래 출력 양식에 맞춰 한국어 존댓말로 정리한다.

## 출력 양식
### 회의 요약
(2~3문장)

### 결정 사항
- 항목별 한 줄

### 할 일 (액션 아이템)
| 담당자 | 할 일 | 기한 |
|---|---|---|

## 규칙
- 입력에 없는 내용은 지어내지 않는다.
- 이름·날짜·숫자는 입력 그대로 사용한다.
- 개인정보(연락처 등)는 결과에 포함하지 않는다.
```

## 예시 입력 (테스트용)
```
오늘 신제품 출시 회의함. 출시일은 7월 15일로 확정. 가격은 다음 회의에서 결정.
민수가 보도자료 초안 금요일까지 쓰기로 함. 디자인은 지영이 다음 주까지.
인스타 광고 예산은 아직 미정.
```

## 기대 출력 (이렇게 나오면 성공)
```
### 회의 요약
신제품 출시 회의를 진행했습니다. 출시일을 확정했고, 가격과 광고 예산은 추후 결정하기로 했습니다.

### 결정 사항
- 출시일: 7월 15일 확정
- 가격: 다음 회의에서 결정

### 할 일 (액션 아이템)
| 담당자 | 할 일 | 기한 |
|---|---|---|
| 민수 | 보도자료 초안 작성 | 금요일 |
| 지영 | 디자인 작업 | 다음 주 |
| 미정 | 인스타 광고 예산 결정 | 미정 |
```

## 응용
- 본인 팀의 진짜 반복 업무로 스킬 만들기(주간보고, 문의 답변, 데이터 표 정리 등) → 이게 **Day 4~5 프로젝트의 재료**가 됩니다.
