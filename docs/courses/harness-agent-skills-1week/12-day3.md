## 강의 노트
### ① Agent Skills
- 정의: AI가 특정 작업을 잘하도록 **미리 정리한 지식·절차 묶음**. 폴더 + `SKILL.md`.
- 작동(점진적 노출): 평소엔 **제목/설명(description)만** 알고 있다가, 관련 요청이 오면 **본문 절차를 펼쳐 읽고** 따른다 → 그래서 description이 "언제 쓰는지"를 정확히 담아야 함.
- 왜 좋은가: ① **재사용**(매번 다시 설명 안 함) ② **일관성**(항상 같은 품질·양식) ③ **공유**(팀 자산).
- 비유: 신입에게 주는 **업무 매뉴얼**. 잘 써두면 누가 해도 같은 결과.
- 구조(템플릿 참고): 상단 `name`/`description`(메타) + 본문 `언제 쓰나 / 절차 / 출력 양식 / 규칙`.

### ② 기술 파일(Technical Files)
- 프로젝트를 굴러가게 하는 파일들:
  - **프로젝트 설명서**(README): 무엇을 만드는지, 어떻게 실행하는지 — AI에게도 맥락이 됨.
  - **설정/구성 파일**: 옵션·환경값(예: 제목, 색, 모드).
  - **`SKILL.md`**: 반복 작업 절차.
  - **샘플 데이터**: 입력 예시(가공/익명).
- 핵심: 이 파일들이 곧 **AI에게 주는 맥락**. 잘 갖출수록 바이브 코딩 품질↑.

## 실습
### 랩 3-1 (75분) — SKILL.md 만들고 호출
1. 각자/팀이 자주 하는 **반복 업무 1개** 선택(예: 주간보고, 회의록 정리, 문의 답변, 데이터 표 정리).
2. [템플릿](./templates/skill-template.md) §4 빈칸을 채워 `my-skills/<이름>/SKILL.md` 작성.
3. 하네스에서 호출:
   ```
   my-skills/<이름>/SKILL.md 의 절차에 따라, 아래 입력으로 결과를 만들어줘:
   (샘플 입력)
   ```
4. 결과가 절차/양식을 따르는지 확인 → 안 맞으면 **SKILL.md의 절차·규칙을 고쳐** 재시도(이게 핵심 연습).

### 랩 3-2 (80분) — 기술 파일 세트
- 팀의 후보 프로젝트로 **기술 파일 세트** 초안 만들기:
  - `README.md`(무엇을/왜/어떻게 실행), 간단 `config`(제목·옵션), `sample-data`(샘플 입력), 그리고 3-1의 `SKILL.md`.
- AI에게 README를 주고 "이 설명대로 프로젝트 뼈대를 잡아줘"까지 시도(내일 빌드의 출발점).

## 산출물

## 과제 (Day 4 준비)
- 팀 문제 **1개로 확정** + [문제 정의 캔버스](./templates/problem-definition-canvas.md) 1~6번 초안 작성해 오기.

## 강사 팁 / 흔한 막힘
- description을 대충 쓰면 스킬이 안 불려옴 → "언제 쓰는지"를 구체적으로 다시.
- 절차가 너무 길고 추상적 → 신입에게 말하듯 **짧은 단계**로 줄이기.
- "스킬이 안 따라져요" → 호출 시 **파일 경로를 명시**하고, 규칙에 "이 절차를 반드시 따르라" 추가.
- 기술 파일을 완벽히 만들려다 시간 초과 → 초안이면 충분, 내일 빌드하며 보완.
