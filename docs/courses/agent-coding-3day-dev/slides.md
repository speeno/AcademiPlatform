# Day 1 — 워크플로우 다지기

---

## 멘탈 모델 + 플랜-퍼스트
- 하네스 = 도구·맥락·루프 / 사람 = 목표·검증
- 큰 변경은 **계획부터** → 검토 → 실행 → diff 리뷰
- `CLAUDE.md`로 컨벤션·테스트·명령 고정

> 랩: 버그 수정 + 기능 추가(테스트로 검증)

<!-- ===== DAY 2 ===== -->
# Day 2 — 스킬 · 서브에이전트 · MCP

---

## Agent Skills 라이브러리
- 반복 작업을 `SKILL.md`로 → 버전관리·팀 공유
- 예: test-writer · commit-message · pr-describer
- 점진적 노출로 필요할 때만 로드

---

## 서브에이전트
- 독립·병렬 작업(코드베이스 조사, 다중 파일)을 **위임**
- 메인 컨텍스트 절약 → 더 길고 복잡한 작업 가능
- 패턴: "관련 코드 조사·요약" → 결과 받고 본 작업

---

## MCP (Model Context Protocol)
- 외부 도구/데이터(예: GitHub)를 에이전트에 **연결**
- 이슈·PR·문서 등 외부 컨텍스트를 워크플로우에 통합
- 설정·토큰은 **공식 문서 기준 + 비밀 관리** 주의

---

# Day 3 — 자동화 + 팀 적용

---

## 자동화 파이프라인
- 코드 리뷰 / 테스트 생성 자동화
- **훅·슬래시 커맨드**: 변경 시 테스트·린트 자동 실행
- **CI 통합**: PR 자동 점검(개념·실습)

---

## 팀 적용 & 표준화
- 스킬셋 + `CLAUDE.md` + 자동화를 **실제 레포에 커밋**
- "한 명이 잘 쓰기" → "팀 워크플로우로 고정"

---

## 안전 (개발 트랙)
- ❌ 토큰·.env·고객 데이터 금지
- ✅ 변경은 **테스트·실행·diff 리뷰** 후 머지
- ✅ 생성 코드 보안·라이선스 점검
