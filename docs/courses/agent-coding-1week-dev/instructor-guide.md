# [D7] 에이전트 코딩 1주 — 강사 운영안 (Day 1~5)

> D1·D3의 기본기·심화를 1~3일에 다지고, Day4~5에서 **오케스트레이션·컨텍스트 엔지니어링·관측·표준화**로 조직 적용까지. 최종 산출물은 **팀 레포에 커밋된 표준 워크플로우**.

## 준비물 (강사)
- 샘플/팀 레포(테스트·CI 골격), [개발자 쿡북](../_shared/prompt-cookbook-dev.md), [D1](../agent-coding-1day-dev/labs.md)·[D3](../agent-coding-3day-dev/labs.md) 랩, [D7 랩](./labs.md)
- MCP/오케스트레이션 데모 환경, 토큰/비밀 취급 안내

> ⚠️ 모델/도구/SDK/MCP의 정확한 명령·설정은 **수업 시점 공식 문서** 기준으로 시연·안내(자료 표기는 개념 위주).

## 운영 핵심 팁
- **검증 강박**: 모든 AI 변경은 테스트·CI·diff 리뷰 후 머지(자동 수락 금지).
- **비밀/보안**: 토큰·`.env`·고객 데이터 금지, MCP 토큰 최소권한·화면공유 주의. (심화: 프롬프트 인젝션·비밀유출 점검)
- **오케스트레이션은 도구**: 항상 여러 에이전트가 이득은 아님 — "분할·병렬·검증"이 가치 있을 때만. 단순 작업은 단일 에이전트.
- **컨텍스트가 비용·품질의 핵심**: 넣을 것/뺄 것을 의도적으로. `CLAUDE.md`·검색·캐싱·압축 활용.
- **표준화 관점**: 산출물은 "한 명의 팁"이 아니라 **레포에 커밋된 팀 표준**(스킬·자동화·문서·게이트).
- 변동 정보는 공식 문서 기준으로 그때그때 확인. 안전: [safety-ethics](../_shared/safety-ethics.md).
